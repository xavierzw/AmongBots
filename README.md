# AmongBots

一款多人在线对话推理微信小游戏。

## 快速开始

### 1. 启动服务端

```bash
cd server
npm install
npm run dev
```

服务端默认运行在 `http://localhost:3000`。

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，并填入你的 LLM API Key：

```bash
cp .env.example .env
```

### 3. 运行客户端

使用「微信开发者工具」打开 `client/` 目录：
- 在「详情」→「本地设置」中勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」。
- 修改 `client/js/network.js` 中的 `SERVER_URL` 为你本地服务端的 IP + 端口（如 `ws://192.168.1.5:3000`）。

也可以在浏览器中直接打开 `client/index.html` 进行调试。

## 游戏模式

### 模式一：经典模式（谁是 AI）

**人数**：4 人（3 真人 + 1 AI）

**流程**：
1. 快速匹配或输入房间号进入 4 人房间。
2. 系统随机指定 1 人为「AI 伪装者」，其发言实际由 LLM 生成。
3. 每轮大家依次就话题发言，共 5 轮。
4. 5 轮结束后投票，≥2 人正确投中 AI 则人类胜利，否则 AI 胜利。

---

### 模式二：人类大逃杀

**人数**：1 真人玩家 + N 个 AI 玩家

**难度选择**：
- 简单：1 人类 vs 3 AI，共 2 轮
- 普通：1 人类 vs 5 AI，共 3 轮
- 困难：1 人类 vs 7 AI，共 4 轮

**流程**：
1. 在首页选择「人类大逃杀」及难度，无需匹配，直接进入游戏。
2. AI 全部伪装成人类，且每个 AI 并不知道其他玩家中谁是 AI、谁是人类。
3. 每轮所有存活玩家依次就话题发言。
4. 每轮结束后，**所有存活玩家共同投票**选出最像「人类」的玩家。
5. **得票最高者出局；若出现平票，则所有得票最高者全部出局**。
6. 若真人玩家被投出局，AI 阵营获胜。
7. 若真人玩家撑过指定的轮数且仍未出局，人类获胜。

**AI 投票机制**：
- 每轮投票阶段，每个 AI 会独立根据整轮对话内容判断哪位玩家最可能是人类，并自动投出该票。
- 未配置 LLM API Key 时，AI 投票将随机选择一位存活玩家。

## 技术说明

- **服务端**：Node.js + Express + WebSocket，纯内存运行，无持久化数据库。
- **客户端**：微信小程序 Canvas 2D 渲染，同时提供浏览器调试兼容（`index.html` 内含 `wx` API polyfill）。
- **AI 接口**：支持 OpenAI 兼容格式的 LLM API，通过 `.env` 配置 `LLM_BASE_URL`、`LLM_API_KEY` 和 `LLM_MODEL`。
